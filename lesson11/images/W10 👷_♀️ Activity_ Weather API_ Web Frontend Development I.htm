    W10 üë∑‚Äç‚ôÄÔ∏è Activity: Weather API

    The purpose of this activity is to introduce you to a weather API that you can consume and apply to your Weather Site Project. This activity is required in order to complete the week's assignment where you will use live, weather data versus placeholder content on the town page weather summary. We will be using the site API specifications and the fetch API to retrieve data in the JSON format. You will sign up for a free API key through the site, look at JSON formatted results and how to consume those results in test application.

    Instructions

    ‚ñ∫ Read What Is an API (Links to an external site.)?‚Äîyou do not need to drill down beyond the one page. You have already been using various APIs so what are they?

    ‚ñ∫ Navigate to OpenWeatherMap (Links to an external site.) and enter your city name in the input box provided and click Search. Click on the appropriate search result for your city/town. Note that some weather data is provided in those results. You may want to switch to C or F results using the buttons in the upper-right corner of the site.

    A city template page will load with the current weather information formatted to the page. You will be using results like these and placing them into your own town page weather summary and forecast. The raw format of the data will be requested in JSON format from which you can extract the named properties that you want to use on your page.

    ‚ñ∫ Click on the API link in the main menu at the top of the page or navigate to OpenWeatherMap: Weather API (Links to an external site.).
    Find the current weather data section, and click on the API doc button.

    The current weather data page explains how to call the current weather data for one location.
    You want to look at a sample response (output) by using the By city ID method provided by the service.

    ‚ñ∫ Scroll down to By city ID and click on the Examples of API calls: link provided. When you click on that link, an API key is added for you in the URL and the results are in returned in a minimized JSON format. Look at the results and try to pick out some name-value pairs  e.g., "description," "scattered clouds"), and try to make sense of the output.
    Some of the results may seem odd but remember that there are different units available to measure temperature such as Kelvin.

    üí° Type "Kelvin to Fahrenheit" as a Google Search to get the Google quick unit converter (UI) to complete a quick conversion test.

    ‚ñ∫ Close that browser tab and click back on the site to get to the Subscribe option OR go to OpenWeatherMap: Price (Links to an external site.). You will be using the Free account which allows no more than 60 calls to the service per minute. You want to be careful as you are testing your work to not preform excessive calls. Note that this service is provided under Creative Commons Attribution-ShareAlike 4.0 International license (CC BY-SA 4.0) (Links to an external site.) and that the data is open and licensed by Open Data Commons Open Database License (ODbL). You need to credit OpenWeatherMap as a weather data course in a visible part of your site.

    ‚ñ∫ Click on the Get API key and Start button in the Free account column.

    ‚ñ∫ Bookmark the How to Start page that loads to keep as a reference.

    ‚ñ∫ Click on the Sign Up button provided and proceed through the guided directions to get an API key (APPID) which you will need to store and keep secure for your use only.

     

    üí° Make sure to record your API key (APPID) as this unique key is required in the URL as a querystring to get/call the service.

    ‚ñ∫ Return to the bookmarked #use page (Links to an external site.) and read the sections about how to use API key in API call and getting accurate API responses along with access limitation warnings.

    ‚ñ∫ In this activity and on the assignment, you will be using Preston, Idaho town in the U.S. with the city id of 5604473. This number is needed in the API call URL. You can find all the city id codes on the openweathermap documentation site. We provide the city codes for the weather site to you in the assignment descriptions going forward.

    Now, with your unique APPID and the CITYID you can test a simple call to the OpenWeatherMap API service. In this activity, you will build a simple HTML document that references JavaScript that uses the fetch API (Links to an external site.) and the fetch() method to get or fetch weather information asynchronously on OpenWeatherMap.

    üí° You will need to bookmark OpenWeatherMap: Current weather data (Links to an external site.) to reference all of the Weather parameters that are available to use in the API JSON response. For example, to get the current weather description, use weather.description

    Requirements

    üé¶ Video and activity: Attend or watch the Virtual Lab demonstration this week.
      Try to complete this assignment before the lab.

    ‚ñ∫ Create a basic web page named "api-test.html" starting with the following markup in the <body>. Make sure this file and its supporting references are found in a "lesson-10" folder. 

    ‚ñ∫ Create a "weatherapi.js" file in your "js" folder and source reference that file before the <body> closing tag.

    <h1>OpenWeatherMap.org API Test</h1>
<p>The current temperature in Preston, Idaho is <span id="current-temp"></span>&deg; F</p>
<hr>
<p>Image Icon Path Test: <span id="imagesrc"></span> &nbsp; <img src="" alt="" id="icon"></p>

<script src="js/weatherapi.js"></script>

    ‚ñ∫ In your "weatherapi.js" file, first create an "apiURL" variable using const that stores the 'https://api.openweathermap.org/...' URL as demonstrated in OpenWeatherMap: How to start (Links to an external site.).

    • Use weather versus forecast data. 
    • The querystring name/value pair settings in the URL should be:
      • a city id for Preston, Idaho,
      • temperature units in Fahrenheit (imperial), and
      • your own APPID.

    üí° The city ID variable at OpenWeatherMap for Preston Idaho is 5604473.

    ‚ñ∫ Use fetch() to request the given apiURL. It is OK to not use a lot of error checking in this activity with fetch(). Convert the string response to a JavaScript object (.json) and print the results to the console in order test and evaluate what is available to use. Use arrow functions.

    const apiURL = "...";
fetch(apiURL)
  .then((response) => response.json())
  .then((jsObject) => {
    console.log(jsObject);
  });

    ‚ñ∫ Run the page via Live Server locally and verify that the everything works. If you have an error, most likely your URL is incorrect and/or you have a bad querystring value or values.

    ‚ñ∫ Open the Inspect DevTools and click on the Console tab. Review and drill into the log results to see what was delivered.

    ‚ñ∫ The current temperature is found under "main" and is named "temp". In your "weatherapi.js" file, write the temperature value to the HTML document using id value of the <span> tag with the id of 'current-temp'.
    This line of code should immediately follow the console.log() statement.

    document.getElementById('current-temp').textContent = jsObject.main.temp;
    üí° It is highly recommended that you not just copy this line of code without explaining it in detail to someone.

    ‚ñ∫ The current weather icon is found under the "weather" array, at index of [0] and is named "icon". This icon is just a preset code value that corresponds to OpenWeatherMap's library of images which is found at https://openweathermap.org/img/w/ (Links to an external site.)

    • Use a variable to store the image address which needs to be concatenated together given the API icon code value result.
    • Use a variable to store the weather description.
    • Assign the <span> tag  with the id of 'imagesrc' the concatenated image variable to to display the full address for testing purposes.
    • Set the src attribute for the <img> with the id of 'icon'.
    • Set the alt attribute for the <img> for accessibility.
    const imagesrc = 'https://openweathermap.org/img/w/' + jsObject.weather[0].icon + '.png';  // note the concatenation
const desc = jsObject.weather[0].description;  // note how we reference the weather array
document.getElementById('imagesrc').textContent = imagesrc;  // informational specification only
document.getElementById('icon').setAttribute('src', imagesrc);  // focus on the setAttribute() method
document.getElementById('icon').setAttribute('alt', desc);
    ‚ñ∫ Test and debug your page until you have output similar to as shown here.
    ‚ñ∫ Work with your Teaching Team and/or the class on any issues.

    Styling the page is optional.

    You will NOT be submitting this activity for assessment.
